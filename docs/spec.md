# 取扱説明書を読むゲーム 仕様書（Draft）

> **注記:** これは初期開発時のドラフト仕様書です。現在の実装状態は [CLAUDE.md](../CLAUDE.md) および [design.md](design.md) を参照してください。

---

## 1. 概要
- 取扱説明書が逐次更新され、ルールが書き換わるアクションゲーム。
- プレイヤーは説明書のバージョンの選択を迫られる。
- 最終的に説明書を投げ捨てる
- 横スクロールの骨格は維持しつつ、説明書の選択でゲーム性が育つ。
- 説明書の選択によって"ゲームを作る"感覚が生まれ、高スコアを狙う。

## 3. ゲーム進行（ラフ）
- Phase A: 初期版の説明書通りに遊ぶ（軽いチュートリアル）。
- Phase B: 説明書更新で操作・危険物が入れ替わる。
- Phase E: 説明書の内容からゲームの方向性を固めていく。

## 4. 主要要素
- 説明書UI: 画面右下に常時表示。更新時に差し替えアニメーション。
- プレイヤー操作: 左右移動・ジャンプが基本。更新でキー変更や制限。
- 危険オブジェクト: 色や形で危険/安全が決まる。
- 更新履歴: 直近数件の説明書更新を閲覧可能（混乱を見える形にする）。
- ルール成長: 2択の積み重ねでプレイスタイルが切り替わり、高スコア向けに育てる。

## 5. ルール更新例
- Ver1.0: スペースでジャンプ / 赤を避ける
- Ver1.1: 青が危険 / ジャンプキーがEに変更
- Ver1.7: 左にしか移動できない
- VerX.X: 「ジャンプ多用を禁止」など個別学習ルール

## 5.1 バージョン選択システム（追加）
- 説明書更新時に2択を提示し、プレイヤーが次のバージョンを選ぶ。
- 選択はゲームプレイに直結し、操作・危険物・UIの挙動が分岐。
- 横スクロールの枠は崩さず、派生ルールを重ねて"別ゲー感"を出す。

## 5.2 ルール成長（デッキ構築に近い発想）
- 2択の選択履歴が「構築内容」として蓄積される。
- 蓄積されたルールの組み合わせでスコアが伸びる設計。
- 横スクロールから別ゲーに派生する軸は「走行ルール系」と「リズム/入力系」を重視する。
- 走行ルール系（派生の核）:
  - オートラン軸: 回避/射撃中心の設計に寄る。
  - 低速精密軸: 足場/位置取りに寄せてプレイ感が変わる。
  - 空中滞在軸: 空中コンボや滞空スコアを伸ばす。
  - 地上加速軸: 地上連続回避で倍率が上がる。
- リズム/入力系（別ゲー感の補強）:
  - 一定テンポ入力で安全扱いや加点が起きる。
  - タイミング良しで反射/無敵などの入力評価が入る。
  - 拍ごとに危険色が切り替わるなど、リズム依存の障害が増える。
- 成長パラメータの例:
  - テンポ: リズム入力や連続行動に強くなる。
  - レンジ: 射撃や遠距離の立ち回りが伸びる。
  - 敵要素: 敵/弾幕が有効になりSTG寄りに派生する。
  - コンボ: 連続回避/連続撃破で倍率が伸びる。
- ルート例:
  - STG寄り: 前方ショットが追加される。
  - RUNNER寄り: オートランになり回避主軸になる。
  - コンボ集中型: 連続成功で高倍率を狙える。

## 6. 説明書の学習仕様（案）
- トリガー: 特定行動の使用頻度が一定割合を超えたら発動。
- 例:
  - ジャンプ率 > 40% → 「ジャンプ禁止」
  - 右移動多用 → 「右移動禁止」
  - 赤回避成功続き → 「赤は安全」などの逆指示

## 7. ギブアップ制 + 説明書投擲
- 任意タイミングでギブアップ可能。
- ギブアップ時に投擲フェーズへ遷移。
- 投擲後、捨てた説明書を"自分で読む"後日談演出を入れる。

### 7.1 投擲操作（決定）
- 右下の説明書UIをドラッグして投げる。
- ドラッグ中にパワーゲージ表示。
- リリース時に投擲（角度 = ドラッグ方向、パワー = ゲージ）。

### 7.2 投擲スコア（決定）
- 軌跡評価はアーチ重視。
- 滞空時間 + 弧の高さを主要評価。
- 速度は減点要素（速すぎると評価低）。
- "ふわっと入る"が最高評価。

## 8. スコア設計（決定）
- 最終スコア = 横スクロール到達スコア(70%) + 投擲スコア(30%)

## 9. 横スクロール（決定）
- 恐竜ゲーム系のエンドレス形式。
- 自動スクロール。
- 障害物の出現頻度が徐々に上がる。
- 説明書更新で操作・危険物が入れ替わる。

## 11. クリア条件
- 説明書の妨害を乗り越え、ゴールに到達。
- もしくは「説明書を破る」演出でゲーム終了。

## 12. 想定ビジュアル/演出
- 白背景+黒文字の説明書が主役。
- 更新時は差分が強調（赤線、取り消し線、手書き風追加）。
- UIが徐々に侵食してくる演出。

## 13. Web実装向けのデータ駆動設計（JSON設計）
- ルール更新やUI挙動をJSONで切り替え可能にする。
- JSONから読み込むもの:
  - 説明書の文面とバージョン
  - 次バージョンの2択候補
  - 操作キー割り当て
  - 危険/安全オブジェクトの定義
  - 学習トリガー条件と変更内容
  - 投擲スコアの重み
  - スコア比率（横スクロール/投擲）
