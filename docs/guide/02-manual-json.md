# 説明書JSON の読み書き

[← 01-genre-system.md](01-genre-system.md) を読んでいることを前提にしています。

---

## 説明書JSONが何をしているか

`src/data/manuals/` 以下のJSONファイルが「説明書」の全内容を定義しています。ゲームの起動時にこれらが自動で読み込まれ、プレイヤーに提示される2択・パラメータ加算・本文テキストがすべてここから来ます。

TypeScriptのコードを書かなくても、**JSONを追加・編集するだけで選択肢を増やせます**。

---

## ファイルの構造

ひとつのJSONファイルは複数のバージョンエントリを持てます。

```json
{
  "id": "action-branch",
  "description": "戦闘・探索系の分岐（開発者メモ）",
  "entries": [
    {
      "key": "2.0-a",
      "version": "2.0",
      "manualText": [
        "ステージに変化が生まれました。",
        "選択してください。"
      ],
      "controls": {
        "jump": "Space",
        "moveLeft": "ArrowLeft",
        "moveRight": "ArrowRight"
      },
      "hazards": {
        "colors": ["red"],
        "safeColors": ["blue"]
      },
      "choices": [
        {
          "label": "ステージに登場するものに個性を加える",
          "next": "3.0-stg",
          "genreParams": { "range": 2, "enemy": 2 }
        },
        {
          "label": "主人公に名前と物語を与える",
          "next": "3.0-rpg",
          "genreParams": { "growth": 3 }
        }
      ]
    }
  ]
}
```

---

## 各フィールドの意味

### エントリ（1バージョン）のフィールド

| フィールド | 必須 | 説明 |
|---|---|---|
| `key` | ─ | マップキー。省略すると `version` の値が使われる |
| `version` | ✅ | 説明書ヘッダーに表示されるバージョン文字列（例: `"2.0"`） |
| `manualText` | ✅ | 説明書の本文。行ごとに配列で書く |
| `controls` | ─ | 操作キーの定義。省略すると前バージョンから継承 |
| `hazards` | ─ | 危険色・安全色の定義。省略すると前バージョンから継承 |
| `choices` | ─ | 次バージョンへの2択。省略または空配列でジャンル収束 |

### 選択肢（choice）のフィールド

| フィールド | 必須 | 説明 |
|---|---|---|
| `label` | ✅ | プレイヤーに表示するテキスト |
| `next` | ✅ | 選択後に遷移するエントリの `key` |
| `genreParams` | ✅ | 加算するパラメータ（例: `{ "tempo": 2, "aerial": 1 }`） |
| `hint` | ─ | 開発者向けメモ（プレイヤーには見えない） |
| `paramMultiplier` | ─ | genreParams への乗数（デフォルト: 1.0）。`1.5` にすると加算値が1.5倍になる |
| `genrePoints` | ─ | ジャンルへの直接ポイント付与（例: `{ "stg": 3, "rpg": 1 }`） |

---

## genreParams がベイズ収束にどう影響するか

各選択肢の `genreParams` は累積されていきます。この累積値がジャンルの「期待中心（thresholds）」に近ければ近いほど、そのジャンルの事後確率が高くなります。

事後確率が 50%を超えるとジャンルが確定します。ブラウザのコンソールで `[BAYES]` ログを確認すると、各ジャンルの確率分布がリアルタイムで確認できます。

---

## 選択肢のラベルにジャンル名を書かない理由

`label` にはジャンル名を直接書きません。「STGにする」「RPG要素を追加する」という書き方は避けます。

これはゲームのコンセプトに直結したルールです。

このゲームの核心は「プレイヤーがジャンルを意識せずに選択し、気づいたらゲームが変わっていた」という体験です。プレイヤーが「今どのジャンルに向かっているか」を意識しながら選択するのではなく、「ゲームの雰囲気や手触り」について直感で選んでいるうちに、結果としてジャンルが生まれる——この発見の驚きがゲームの面白さの中心にあります。

ラベルがジャンル名を明示してしまうと、選択がただの「ジャンル選択メニュー」になります。

```
✗ "シューティングゲームにする"  → プレイヤーはジャンルを選んでいる
✓ "ステージに登場するものに個性を加える" → プレイヤーはゲームの変化を選んでいる
```

`hint` フィールドは開発者向けのメモ欄です。「STG方向の選択」のような補足はここに書いてください。プレイヤーには表示されません。

## 新しい選択肢を追加する最小手順

**ケース1: 既存のブランチに選択肢を追加したい**

`action-branch.json` などを開いて、対象エントリの `choices` 配列に追記します。

```json
{
  "label": "ステージを空へ広げる",
  "next": "3.0-aerial",
  "genreParams": { "vertical": 3, "aerial": 1 },
  "hint": "aerial_stg 方向"
}
```

ファイルを保存して開発サーバーをリロードすれば反映されます。コードの変更は不要です。

**ケース2: 新しいブランチファイルを作りたい**

`src/data/manuals/TEMPLATE.json` をコピーして新しいファイルを作成します。`id`・`key`・`choices.next` が全ファイルを通じてユニークになるよう注意してください。

## ゲームの挙動を変えるには

`runtimeOverrides` フィールドで、そのバージョン期間中だけ有効なゲームパラメータを上書きできます。

```json
{
  "key": "3.0-fast",
  "version": "3.0",
  "manualText": ["スピードが上がりました。"],
  "controls": { "jump": "Space", "moveLeft": "ArrowLeft", "moveRight": "ArrowRight" },
  "hazards": { "colors": ["red"], "safeColors": ["blue"] },
  "runtimeOverrides": {
    "scrollSpeed": 320,
    "gravity": 1400
  },
  "choices": []
}
```

よく使う上書きフィールド：

| フィールド | 意味 |
|---|---|
| `scrollSpeed` | スクロール速度（px/秒） |
| `gravity` | 重力加速度 |
| `playerMaxHp` | 最大HP（`hp` フィーチャー有効時） |
| `scrollDirection` | `"vertical"` で縦スクロールに切り替え |
| `environment` | `"ocean"` / `"space"` など背景環境の切り替え |

## よくあるミスと確認方法

| 症状 | 原因 |
|---|---|
| 選択肢を選んでも何も起きない | `next` のキーが存在しないか、タイプミス |
| ずっとジャンルが確定しない | 累積値がどのジャンルの期待中心にも近づいていない。`[BAYES]` ログで事後確率を確認 |
| バリデーションエラーがコンソールに出る | `next` の参照先キーが存在しない |

開発中は起動時に自動バリデーションが走ります。ブラウザのコンソールで以下が出れば正常です。

```
[ManualDeck] Validation OK (42 entries, 0 errors, 2 warnings)
```
