# ゲーム全体像ガイド

このシリーズは「このプロジェクトの仕組みを順番に理解する」ための入口ドキュメントです。コードをゼロから読む前に、まずここで全体像をつかんでください。

---

## このゲームが何をするか

横スクロールの簡単なアクションゲームとしてスタートします。プレイヤーは障害物をかわしながら走るだけです。

しかし、ゲームの途中で「説明書」が更新されます。プレイヤーは提示される2択を選び続けることで、ゲームそのものが変わっていきます。最初は横スクロールだったゲームが、シューティングになったり、RPGになったり、リズムゲームになったりします。

最終的に「もう十分だ」と感じたとき、プレイヤーは説明書を画面に投げます。それがゲームの完成です。

---

## 4つの主要概念

このゲームを支える仕組みは4つの概念で成り立っています。

**説明書（Manual）**
プレイヤーが読み、選択肢を選ぶもの。JSONファイルで定義されていて、「次はどちらのルールにするか」という2択を提示します。

**ジャンルパラメータ（GenreParam）**
選択肢を選ぶたびに蓄積される数値。`tempo` や `enemy` など12軸あります。プレイヤーが選んだ方向性が数値として積み上がっていきます。

**ジャンル（Genre）**
パラメータが一定の閾値に達したとき確定する「ゲームの種類」。STG・RPG・リズムゲームなど22種が定義されています。ジャンルが確定すると、見た目・操作感・スコア計算がまとめて切り替わります。

**フィーチャー（Feature）**
ジャンルが確定したときに有効・無効になる個別の機能。`shoot`（弾を撃てる）や `hp`（HPがある）のような単位です。ジャンルは「どのフィーチャーを使うか」の組み合わせで定義されています。

---

## 4つの概念の関係

```
プレイヤーが選択肢を選ぶ
        ↓
  GenreParam が蓄積される
  （12軸の数値が積み上がる）
        ↓
  閾値を超えたジャンルが確定
        ↓
  そのジャンルの Feature が有効になる
  （見た目・スポーン・スコア式も切り替わる）
```

説明書の選択肢がパラメータを動かし、パラメータがジャンルを決め、ジャンルがフィーチャーを決める。この一方向のデータフローが基本です。

---

## ゲームの進行フェーズ

ゲームは以下の順番で進みます。

```
tutorial（チュートリアル）
  横スクロールをそのまま体験する。操作を覚える段階。
        ↓
updating + playing（説明書更新とプレイ）を繰り返す
  説明書が更新されるたびに2択が提示される。
  選んだ後はそのルールでゲームをプレイする。
  この繰り返しでジャンルパラメータが蓄積されていく。
        ↓
genreLocked（ジャンル確定）
  パラメータが閾値に達し、ジャンルが確定する。
  見た目・操作・スコア計算がそのジャンルに切り替わる。
        ↓
throwing（投擲）
  説明書をドラッグして画面に投げる。「作り終えた」宣言。
  投げ方（滞空時間・弧の高さ）がスコアの一部になる。
        ↓
ending（エンディング）
  ジャンルごとのエンディングメッセージと最終スコアが表示される。
```

## ゲームを構成するファイルの場所

コードとデータが明確に分かれています。

| 役割 | 場所 |
|---|---|
| ジャンル定義（閾値・フィーチャー・スコア式） | `src/data/genres/*.json`（1ジャンル1ファイル） |
| 説明書の選択肢・本文 | `src/data/manuals/*.json` |
| ジャンルの見た目（背景・プレイヤー描画） | `src/genres/` |
| フィーチャーの動作ロジック | `src/game/systems/` |
| ゲームの物理・衝突・スクロール | `src/game/sideScroller.ts` |

新しいジャンルを追加したいとき、新しい選択肢ルートを作りたいとき——多くの場合はJSONを書くだけで済む設計になっています。
