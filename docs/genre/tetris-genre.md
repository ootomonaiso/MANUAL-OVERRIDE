# テトリスジャンル実装ドキュメント

## 概要

MANUAL-OVERRIDEゲームに「テトリス」を新しいジャンルとして追加しました。
プレイヤーが説明書の選択肢を積み重ねることで、横スクロールゲームからテトリスパズルゲームへと変貌します。

## アーキテクチャ

### ファイル構成

```
src/
├── data/
│   ├── config/genres.json          # ジャンル定義（tetris追加）
│   └── manuals/tetris-branch.json  # テトリス分岐の選択肢（12エントリ）
├── genres/
│   ├── TetrisPlugin.ts             # 視覚テーマ（暗色系・グリッド線・Tブロック）
│   └── index.ts                    # プラグイン登録
├── game/systems/
│   ├── TetrisFeature.ts            # ゲームロジック
│   └── index.ts                    # フィーチャー登録
└── domain/types.ts                 # ManualTheme型に'tetris'追加
```

## ジャンル収束条件

| パラメータ | 閾値 |
|-----------|------|
| `combo`   | 4以上 |
| `craft`   | 4以上 |

両方のパラメータが閾値を超えるとテトリスジャンルに収束します。

### 収束パスの検証

action-branchからの経路（`1.0 → 2.0-a → 3.0-a-grow → 4.0-a-craft → 2.0-t`）を起点として、
テトリス分岐内の全16パスが `combo >= 4 AND craft >= 4` に到達することを確認済みです。

## ゲーム仕様

### グリッド

- 10列 × 20行
- 各セルは色（`string`）または空（`null`）

### テトリミノ（7種類）

| ID | 色 | 形状 | 回転状態数 |
|----|----|------|-----------|
| I | `#00f0f0` | 直線 | 2（水平/垂直） |
| O | `#f0f000` | 正方形 | 1（回転不変） |
| T | `#a000f0` | T字 | 4 |
| S | `#00f000` | S字 | 2 |
| Z | `#f00000` | Z字 | 2 |
| J | `#0000f0` | L字 | 4 |
| L | `#f0a000` | 逆L字 | 4 |

I/S/Zピースは2状態のみですが、`% 4` 互換性のために4エントリ配列として格納しています。

### 7-Bagランダム

7種類のピースをシャッフルした「バッグ」を1セットとして使用し、
空になったら次のセットを生成します。これによりピース出現が公平になります。

### 操作方法

| キー | 動作 |
|------|------|
| ← → | 左右移動 |
| ↑ | 回転（時計回り） |
| ↓ | ソフトドロップ（加速落下） |
| Space | ハードドロップ（即座に落下） |

### スコア計算

| 項目 | 点数 |
|------|------|
| 1ライン消去 | 100点 |
| 2ライン消去 | 300点 |
| 3ライン消去 | 500点 |
| 4ライン (Tetris!) | 800点 |
| ソフトドロップ | 1セルあたり +1点 |
| ハードドロップ | 1セルあたり +2点 |

最終スコア式: `combo * 300 + survivedSec * 5 + distance * 0.1`

**注意**: テトリスモードでは `scrollSpeed = 0` であるため `distance` 項は常に0になります。

### コンボシステム

- `currentStreak`: 連続ライン消去の累積ライン数
- ラインを消去するとストリークに消去したライン数が加算される
- ラインを消去せずにロックするとストリークが0にリセットされる
- `world.setCombo()` / `world.resetCombo()` でゲーム全体に通知

### 特殊機能

- **ゴーストピース**: 落下先の予測位置を半透明（alpha=0.2）で表示
- **ロックディレイ**: 接地後1.5秒間は移動・回転可能（最大15回リセット可能）
- **ウォールキック**: 壁際で回転失敗時に水平方向 ±1, ±2 マスシフトして再試行
  - Iピースは垂直方向のキックも試行
  - 簡易実装（完全なSRS準拠ではない）

## 説明書分岐

### tetris-branch.json

12のエントリ（バージョン 2.0-t から 6.0-t）で構成されます。
各選択肢は `combo` と `craft` パラメータを加算し、テトリスジャンルへの収束を促します。

### 統合パス

既存の分岐木から以下の2箇所でテトリスへ分岐可能になりました：

1. **action-branch.json** `4.0-a-craft` の選択肢「ブロックを積み重ねてラインを完成させる」→ `2.0-t`
2. **advanced-branch.json** `10.0-puzzle` の選択肢「ブロックを積み重ねるパズルへ」→ `2.0-t`

## 実装上の注意点

1. `TetrisFeature`は`preUpdate`でプレイヤー移動を無効化（`vx = 0, vy = 0`）
2. `scrollSpeed = 0` でスクロールを停止
3. `tetris_mode`フィーチャーIDが有効なときにのみ起動
4. 内部関数はテスト用にexportされています
5. `scrollDirection`と`scrollSpeed`は`onInit`で保存し、`onDisable`で復元
6. `onManualUpdated`呼び出し時も保存値を上書きしない（`firstInit`フラグ）
7. 全メソッド（`preUpdate`/`update`/`render`）でフィーチャーフラグガードあり
8. ゴーストピース描画は`ctx.save()`/`ctx.restore()`で`globalAlpha`を保護

## レビュー履歴

### Iteration 1 (3 HIGH, 7 MEDIUM, 5 LOW)
- **HIGH-1**: コンボリセットバグ（`world.resetCombo()`がデッドコード）→ 修正
- **HIGH-2**: `baseScrollSpeed`プロパティ不存在 → `savedScrollSpeed`で修正
- **HIGH-3**: 分岐パス収束不全 → 閾値を `{combo:4, craft:4}` に変更

### Iteration 2 (1 HIGH, 4 MEDIUM)
- **HIGH-1**: `onManualUpdated`が保存値を上書き → `firstInit`フラグで修正
- **MEDIUM-1**: `update()`のフィーチャーフラグガード未実装 → 追加
- **MEDIUM-3**: ゴーストピースの`globalAlpha`保護不足 → `save()`/`restore()`追加
- **MEDIUM-4**: コンボセマンティクスのコメント不正確 → 修正

### Iteration 3 (0 HIGH, 2 MEDIUM, 6 LOW)
- HIGHなしでレビュー終了
- 残るMEDIUM: ウォールキックSRS非準拠（簡易実装）、カラー定義のDRY違反
